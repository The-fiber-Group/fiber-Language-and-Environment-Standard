Language Concepts

Program Structure

  1. An ASC program consists of a number of entities which contain data. An entity here is not described as an object because entities in ASC programs are not required to hide any internal information they may contain.

  2. Each entity has a representation, a type, and a name.

  3. Representations determine the precision, physical size, and mutability of the data contained by an entity. Constant (non-mutable) entities are used to define the behavior of a program. Mutable entities are used to hold data as operations are performed on it by the program.

  4. Types determine the set of possible data values that can be held by an entity and the operations that are available for that set. Types are used to verify that the behavior of the program is correct, or rejected as incorrect according to requirements established by type definitions.

  5. Names determine the contexts within which an entity is accessible. Names are used during a program's execution to locate both the constant entities containing transformations that the program will perform, and the mutable entities which hold the data being transformed.

  6. A number of primitive entities are provided by the ASC language for creating programs. They are...

    • symbols
    • types
    • variables
    • operators
    • continuations
    • functions
    • modules

  7. Entities are instantiated in a program via declaration and definition.

  8. An entity declaration establishes its name and type.

  9. An entity definition establishes its representation and the initial data that it will contain when

  10. An entity declaration is required before the definition of that entity occurrs.

  11. The complete definition of an entity may be established by multiple separate expressions.

Program Execution

  1. Program execution is facilitated by the evaluation of expressions in an environment of data objects.

  2. The evaluation of expressions may produce some value or effect on the data held by mutable entities.

  3. There are three discrete stages in program execution which occurr in the order of, inspection, translation, then interpretation.

  4. Each stage must be completed before the next can occurr.

  5. No stage is required to be completed in a single event.

  6. The execution process may be applied individually to parts of a program, or uniformly to an entire program. The granularity of execution is implementation dependent.

  7. Program execution begins when an implementation is provided a program to execute through one or more streams containing source text.

Streams

  1. A stream is an abstract data buffer containing traversable ASCII or Unicode encoded text.

  2. Streams are only required to be traversable in a forward direction so that the source text may be read in the order it was written.

  3. Streams may support random stream position seeking.

  4. The contents of a stream are not required to be persistent beyond a single reading.

  5. A stream may be derived from the contents of a source file containing ASCII or Unicode encoded source text.
