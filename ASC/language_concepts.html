Language Concepts

  1. An ASC program consists of a number of entities which contain data. An entity here is not described as an object because entities in ASC programs are not required to hide any internal information they may contain.

  2. Each entity has a representation, a type, and a name.

  3. Representations determine the precision, physical size, and mutability of the data contained by an entity. Constant (non-mutable) entities are used to define the behavior of a program. Mutable entities are used to hold data as operations are performed on it by the program.

  4. Types determine the set of possible data values that can be held by an entity and the operations that are available for that set. Types are used to verify that the behavior of the program is correct, or rejected as incorrect according to requirements established by type definitions.

  5. Names determine the contexts within which an entity is accessible. Names are used during a program's execution to locate both the constant entities containing transformations that the program will perform, and the mutable entities which hold the data being transformed.

  6. A number of primitive entities are provided by the ASC language for creating programs. They are...

    • syntax
    • symbols
    • numbers
    • types
    • variables
    • operators
    • continuations
    • functions
    • modules

Syntax

  1. ASC syntax is composed of individual tokens which consist of contiguous series of characters.

  2. Tokens are delimited by any blank or non-printable characters including...

    • ASCII Control Characters (in a range from Null to Unit Separator)
    • Space
    • Unicode point 0081
    • Unicode point 008D
    • Unicode point 008F
    • Unicode point 0090
    • Unicode point 009D
    • Unicode point 00A0
    • Unicode point 00AD
    • Unicode range from 2000 to 200F)
    • Unicode point 2028
    • Unicode point 2029
    • Unicode range from 205F to 206F

  3. Tokens are divided into two classes, symbols and ternimals. Symbols are used to name entities. Ternimals are used to compose and delimit expressions.

Symbols

  1. Symbols consist of a contiguous series of alphanumeric characters and the ASCII dash character.

  2. Alphanumeric characters are defined as the ASCII code point ranges 30-39, 41-5A, 61-7A, and any Unicode points that fall under the Unicode standard definition of Writing Systems.

  3. The ASCII dash character is allowed in a symbol at any point where it is not the first or last character in that symbol.

  4. Symbols are case insensitive. The ASCII code points 41-5A and 61-7A are considered lexically equivalent for the purposes of the ASC language. The equivalence is not general across all points, rather each individual point from the first range is equivalent to the point from the corresponding position in the second range. For example, the symbols symb, SYMB, and sYmB are considered the same symbol.

  5. In the Unicode Latin-1 Supplement block, the code point pairs are considered lexically equivalent in the following ranges...

    00C0-00D600E0-00F6
    00D8-00DD00F8-00FD

    For example, the characters Æ and æ, and the characters Ý and ý are each considered lexically equivalent.

  6. In the Unicode Latin Extended-A block, in the code point range U+0100-U+017E, each even-numbered code point is considered lexically equivalent with the immediately following odd-numbered code point. For example, the characters Ă and ă are equivalent.

Terminals

  1. Terminals consist of a contiguous series of non-alphanumeric characters including the ASCII dash character.

  2. Non-alphanumeric characters are defined as the ASCII code points 21-2F, 3A-40, 5B-60, 7B-7E, and any Unicode points which fall under the Unicode standard definition of General Punctuation.

  3. Unlike symbols, the ASCII dash character is accepted at any point within a terminal.
