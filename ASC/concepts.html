General Concepts

Execution

  1. Program execution is facilitated by the evaluation of expressions in an environment containing data objects.

  2. The evaluation of expressions may produce some value or effect on the data objects.

  3. There are three discrete stages in program execution which occur in the order, inspection, translation, then interpretation.

  4. Each stage must be completed before the next can occur.

  5. No stage is required to be completed in one continuous interval.

  6. The execution process may be applied individually to parts of a program, or a whole program. The granuarity of the application is implementation dependent.

  7. Program execution begins when an implementation is provided a program to execute through one or more streams containing source text.

Streams

  1. A stream is an abstract data buffer containing traversable text.

  2. Streams are required to be traversable in a forward direction so that the source text may be read in the order it was written.

  3. Streams may support random position seeking.

  4. The contents of a stream are not required to be persistent beyond a single traversal.

  5. A stream may be derived from the contents of an ASCII or Unicode encoded source file containing ASC syntax.

  6. Only one stream may be derived from a single source file and it must contain exactly the contents of that file.

  7. If a stream is derived from a source file, that stream will be assigned the same name as the file it was derived from.

  8. The design of a stream that fulfills the required behavior is implementation dependent.

Program Inspection

  1. The inspection stage shall be responsible for reading source text from all streams provided to the implementation and for verifying that a program is well-formed.

  2. A well-formed program shall conform to the syntactic and semantic constraints which are specified in this document.

Program Translation

  1. The translation stage shall be responsible for translating the source text into an interpretable format.

  2. Any number of translations to any interpretable format may take place at this stage.

  3. The translation stage is not required to be reversible. The interpretable format produced by this stage is only required to preserve the information necessary to carry out all program behavior and effects specified by the source text resulting from the inspection stage.

Cached Files

  1. The interpretable representation of a program or part of a program emitted by the translation stage may be stored and used at a later point. This stored representation is called a cached file.

  2. Depending on which parts of a program are translated and stored together, a cached file may be reused in multiple programs.

  3. The method with which a cached file is stored and possibly reused is implementation dependent.

Program Interpretation

  1. The interpretation stage shall be responsible for allocating the computing resources required to carry out the program's behavior and effects and for initiating the execution of that program.

  2. The interpretation stage must only produce the behavior and effects that are recorded in the interpretable representation of the program.

  3. The means through which the behavior and effects of a program are carried out are implementation dependent.

Data Objects

  1. A program is executed through the manipulation of data objects.

  2. All data objects shall be associated with a single name and a single data type for the duration of their existence.

  3. There are two classes of objects, variables and definitions. All data objects shall be classified as either a variable or a definition.

  4. Variable objects may be read and manipulated.

  5. Definition objects may be read, but not manipulated.

  6. The definition objects are...

    • data type definitions
      • primitive
      • abstract
      • composite
    • program definitions
      • functions
      • continuations
      • modules

  7. The variable objects are...

    • natural numbers
    • integers
    • rational numbers
    • boolean values
    • object locations
    • compound objects
